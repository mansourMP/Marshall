# meeting-to-execution

A Codex skill for turning discussions into execution artifacts: task lists, owner maps, decision recaps, risk lists, and follow-up drafts.

## What it provides
- A structured discussion-to-action workflow in `SKILL.md`
- References for output formats, review rules, and follow-up patterns
- A practical way to avoid vague recap notes

## Good fit for
- operations and program managers
- founders
- chiefs of staff
- teams that rely on messy chat or note-based coordination

## Skill contents
- `SKILL.md`
- `agents/openai.yaml`
- `references/output-patterns.md`
- `references/review-rules.md`
- `references/follow-up-workflows.md`

## Install
Copy the `meeting-to-execution` folder into `~/.codex/skills/`.

## Example use
`Use $meeting-to-execution to turn this transcript into a decision recap, task list, and follow-up draft with clear owners and dates.`
