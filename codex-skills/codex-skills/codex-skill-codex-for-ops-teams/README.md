# codex-for-ops-teams

A Codex skill for practical operations work: inbox triage, follow-up drafting, summaries, task breakdown, handoff prep, and approval-aware recurring workflows.

## What it provides
- A concrete ops workflow in `SKILL.md`
- Guidance for classifying work as read-only, draft-first, approval-gated, or blocked
- Concrete ops use cases in `references/use-cases.md`
- First-week rollout guidance in `references/first-week-workflows.md`
- Review boundary guidance in `references/review-boundaries.md`

## Good fit for
- operations managers
- coordinators and chiefs of staff
- founders handling operational inboxes and follow-ups
- admin and support-heavy internal teams
- mixed-technical teams that need structured, reviewable outputs

## Good use cases
- triaging email, Slack, or ticket queues
- drafting reminders, updates, and escalation messages for review
- turning rough notes into owners, tasks, and deadlines
- preparing daily or weekly operating summaries
- defining what an ops team may automate versus what must stay human-reviewed

## Skill contents
- `SKILL.md`
- `agents/openai.yaml`
- `references/use-cases.md`
- `references/first-week-workflows.md`
- `references/review-boundaries.md`

## Install
Copy the `codex-for-ops-teams` folder into `~/.codex/skills/`.

## Example use
`Use $codex-for-ops-teams to triage this inbox, draft follow-ups for the urgent items, and turn the rest into a clear ops action list with owners and approval gates.`
