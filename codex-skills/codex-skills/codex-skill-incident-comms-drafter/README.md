# incident-comms-drafter

A Codex skill for drafting operational incident communications with disciplined certainty, short status updates, and explicit review boundaries.

## What it provides
- An incident communication workflow in `SKILL.md`
- References for update patterns, approval rules, and post-incident follow-up
- A trust-first approach that avoids invented root cause or over-commitment

## Good fit for
- operations teams
- support leads
- engineering managers coordinating incident comms
- founders handling early incident response

## Skill contents
- `SKILL.md`
- `agents/openai.yaml`
- `references/incident-update-patterns.md`
- `references/approval-boundaries.md`
- `references/post-incident-follow-up.md`

## Install
Copy the `incident-comms-drafter` folder into `~/.codex/skills/`.

## Example use
`Use $incident-comms-drafter to draft an internal update, a customer-safe holding message, and a post-incident follow-up from these notes.`
