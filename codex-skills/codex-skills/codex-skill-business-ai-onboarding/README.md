# business-ai-onboarding

A Codex skill for onboarding non-technical and mixed-technical teams to AI workflows. It treats adoption as an operating-model problem: trust boundaries, approvals, first-week rollout sequencing, and role-specific use cases.

## What it provides
- An operational onboarding guide in `SKILL.md`
- First-week rollout guidance
- Approval and trust setup guidance
- Role-based use cases for teams such as accounting, operations, admin, and wholesale market workflows

## Good fit for
- Internal AI rollout planning
- Non-technical team onboarding
- Approval and trust-policy setup
- Department-specific adoption plans

## Skill contents
- `SKILL.md`
- `agents/openai.yaml`
- `references/first-week-rollout.md`
- `references/approval-trust-setup.md`
- `references/role-based-use-cases.md`

## Install
Copy the `business-ai-onboarding` folder into `~/.codex/skills/`.

## Example use
`Use $business-ai-onboarding to design a first-week rollout plan for a non-technical operations team using Codex.`
