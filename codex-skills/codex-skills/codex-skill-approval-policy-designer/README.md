# approval-policy-designer

A Codex skill for designing practical approval policies for agent systems. It helps teams decide what should always ask, what can auto-run, and what should remain blocked.

## What it provides
- A clear approval-policy workflow in `SKILL.md`
- A practical risk matrix in `references/risk-matrix.md`
- Rollout guidance for strict, balanced, and experimental modes
- Example policies for founders, ops teams, and non-technical users

## Good fit for
- agent builders
- teams deploying local or connector-enabled agents
- trust and approval design
- rollout planning for non-technical users

## Skill contents
- `SKILL.md`
- `agents/openai.yaml`
- `references/risk-matrix.md`
- `references/rollout-modes.md`
- `references/examples.md`

## Install
Copy the `approval-policy-designer` folder into `~/.codex/skills/`.

## Example use
`Use $approval-policy-designer to help me define what my agent can do automatically, what must always ask first, and what should stay blocked.`
